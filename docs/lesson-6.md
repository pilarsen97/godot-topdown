# Занятие 6 — Слои коллизий, группы, чистка проекта

> Игра уже играется: герой ходит, имп догоняет, меч сам бьёт каждые
> 2 секунды. Сегодня доводим до состояния «не стыдно показать»:
> заводим слои коллизий, чтобы всё было корректно, и подчищаем хвосты.

## Что узнаем (теория)

### Слои и маски физики
У каждого CollisionShape2D есть две настройки:
- **`collision_layer`** — «**на каких слоях я живу**».
- **`collision_mask`** — «**кого я вижу/с кем сталкиваюсь**».

Это два независимых битовых набора. Например:
- Player живёт на слое `player`, видит `world` и `enemy`.
- Imp живёт на слое `enemy`, видит `world` и `player`.
- Стены живут на `world`, не видят никого (они статичны).

**Без слоёв** все на слое 1, всё со всем сталкивается, ничего нельзя
отфильтровать. Когда заведутся пули — они начнут сталкиваться с самим
игроком, который их и выпустил. Поэтому **с самого начала** учим
называть слои.

### Имена слоёв в Project Settings
В `Project Settings → Layer Names → 2D Physics` ставим имена:
```
1 = player
2 = enemy
3 = player_hitbox
4 = enemy_hitbox
5 = world
```
После этого в инспекторе вместо «1, 2, 3» видим понятные названия —
сильно упрощает жизнь.

### Группы — это не слои
Дети часто путают:
- **Слой** — для **физики** (с кем сталкиваюсь).
- **Группа** — для **поиска по дереву** (кто это).
Они независимые: можно быть в группе `player` и на любом слое.

### `_physics_process` vs `_process`
Ещё раз, потому что важно:
- `_physics_process(delta)` — фиксированно 60 раз в секунду. Сюда
  пишут **всю физику**: `velocity`, `move_and_slide()`, проверку столкновений.
- `_process(delta)` — каждый кадр (может быть 30, 60, 144 в секунду).
  Сюда пишут **визуал**: анимации, эффекты, UI.

В `player.gd` и `imp.gd` сейчас движение в `_process` — это **бажная
наивная версия**. На ветке `refactor/lesson-1` будет показан правильный
вариант.

## Что делаем (практика)

### Шаг 1. Имена слоёв
Project Settings → Layer Names → 2D Physics:
- Слой 1: `player`
- Слой 2: `enemy`
- Слой 3: `player_hitbox`
- Слой 4: `enemy_hitbox`
- Слой 5: `world`

### Шаг 2. Назначить слои сущностям
- `player.tscn` → CharacterBody2D Player → `collision_layer = player` (1).
  `collision_mask` пока не трогаем (для столкновений со стенами,
  если когда-нибудь добавятся).
- `imp.tscn` → `collision_layer = enemy` (2). `collision_mask` тоже пока 0.

Сейчас, на наивной версии, маски пустые — Player и Imp проходят сквозь
друг друга. Это намеренно: мы пока про **схему**, а урон будет на
будущих занятиях.

### Шаг 3. Камера: лимиты
Открыть Player → Camera2D. В инспекторе раздел **Limit**:
поставить `Left=0, Top=0, Right=примерно ширина уровня в пикселях,
Bottom=примерно высота`. Тогда камера не «выезжает за карту», когда
игрок у края.

### Шаг 4. Чистка
- Удалить из скриптов закомментированный код.
- Убрать пустые `func _ready() -> void: pass` и `func _process(delta): pass`,
  если в них реально ничего не делается. (Объяснить детям: это не
  стыдно — это аккуратность.)
- В `.gitignore` добавить:
  ```
  .DS_Store
  /build/
  /export/
  ```

### Шаг 5. Большой запуск
F5. Должно быть:
- Камера не выезжает за карту.
- Имп догоняет, проходит сквозь игрока (это норма для наивной версии).
- Каждые 2 секунды меч бьёт.

Коммит:
```
git add .
git commit -m "lesson-6: collision layers + camera limits + cleanup"
```

## Что НЕ так в этой версии (специально)
Эта серия `lesson-N` — **наивная**. Дети должны её сделать, чтобы
почувствовать «как пишут не задумываясь». Параллельно есть серия
`refactor/lesson-N` — там же те же занятия, но **с правильной
архитектурой**:
- движение в `_physics_process`, а не `_process`;
- типы и `@export` у всех чисел;
- кэш игрока в `_ready()`, а не поиск каждый кадр;
- `World.tscn` как точка входа, в нём `Level + Player`;
- враги внутри `Level`, не в `World`;
- спавн атаки через **EventBus** (глобальные сигналы), а не через
  `player.get_parent().add_child(...)`.

Сравните `git diff lesson-6 refactor/lesson-6` — будет хорошим
домашним заданием для самостоятельных детей.
