# Занятие 4 — Способность атаки: визуал и сцена

> Imp бегает за игроком, но на нас никак нельзя ответить. Сегодня сделаем
> «оружие» — отдельную сцену со спрайтом меча, которую потом можно будет
> спавнить рядом с игроком.

## Что узнаем (теория)

### Зачем способность — это отдельная сцена
В играх часто появляются и исчезают объекты: пуля, взрыв, звук удара,
огненный шар. Они живут несколько секунд, делают свою работу и удаляются.
**Каждый такой объект — отдельная сцена**:
- легко переиспользовать (огненный шар у мага и у босса один и тот же);
- легко дать художнику отдельный файл для редактирования;
- сцена сама знает, как себя анимировать и удалять — никто снаружи не
  должен в это лезть.

### `Node2D` как корень
Когда сцена не персонаж, а просто «эффект на карте» — корнем берут
**Node2D**. Это лёгкий узел с координатами, без физики.

### `instantiate()` и `add_child()`
Чтобы появилась копия сцены в игре, нужно:
```gdscript
var instance = scene.instantiate()
parent.add_child(instance)
instance.global_position = where_to_spawn
```
Это и есть «спавн». Сегодня попробуем спавнить вручную — без таймера.

## Что делаем (практика)

### Шаг 1. Спрайт меча
Положить `weapon_regular_sword.png` в `assets/textures/weapons/`.

### Шаг 2. Сцена `attack_ability.tscn`
1. New Scene → **Node2D**, имя `AttackAbility`.
2. Добавить дочерний **Sprite2D**, перетащить в его `texture` файл меча.
3. Если меч слишком большой/маленький — поправить `scale` на Sprite2D.
4. Сохранить в `src/entities/player/abilities/attack/attack_ability.tscn`.

### Шаг 3. Проверить вручную
1. Открыть `level_01.tscn`.
2. Правой кнопкой → Instantiate Child Scene → `attack_ability.tscn`.
3. Поставить меч рядом с игроком.
4. F5 — меч просто стоит на полу. Это нормально: пока что мы не сделали
   ни анимации, ни кода.

## Куда положили способность и почему
`src/entities/player/abilities/attack/` — длинно, но логично:
- `entities/player/` — это то, что принадлежит игроку;
- `abilities/` — папка для всех его способностей;
- `attack/` — конкретная способность.

Когда появится магия — будет `abilities/fireball/`. Когда появится дэш —
`abilities/dash/`. Папки рядом, всё в одном месте.

## Что НЕ делаем сегодня
- Не пишем скрипт. Сегодня сцена — это только «арт» + узлы.
- Не привязываем атаку к игроку. На следующем уроке заведём контроллер,
  который будет автоматически создавать копии этой сцены.

## Подводные камни
- Если меч прозрачный/розовый — текстура не импортирована. Перетащи
  её в Godot заново.
- Если меч огромный — у спрайта по умолчанию фильтр texture filter
  включает сглаживание. Для пиксель-арта в `Project Settings → Rendering →
  Textures` ставим `Default Texture Filter = Nearest`.
