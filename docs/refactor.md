# Серия `refactor/lesson-N` — те же занятия, но с правильной архитектурой

Основная серия `lesson-1` … `lesson-6` — это «как пишут, не задумываясь».
Дети пройдут её, чтобы прочувствовать, **почему** что-то ломается на масштабе.

Параллельно есть серия `refactor/lesson-N` — те же шесть занятий,
но с правильным кодом и архитектурой **с самого первого урока**.

## Главные отличия refactor-серии (нарастающим итогом)

| Lesson | Что добавляется по сравнению с наивной |
|---|---|
| **refactor/lesson-1** | `_physics_process` вместо `_process` для движения; типы (`: float`, `: Vector2`); `@export var max_speed`; именованные слои коллизий с самого начала; `collision_layer = player` у Player. |
| **refactor/lesson-2** | Без изменений в коде — добавляется уровень и инстанс игрока. Player всё ещё внутри Level (потом разнесём). |
| **refactor/lesson-3** | У Imp — кэш ссылки на игрока в `_ready()` (никаких поисков по группе каждый кадр). Типы. `_physics_process`. |
| **refactor/lesson-4** | Без изменений в коде. |
| **refactor/lesson-5** | AttackController **не лезет в `player.get_parent()`** — а эмитит сигнал в **EventBus** (autoload). Заводим `src/autoload/event_bus.gd`. |
| **refactor/lesson-6** | **`World.tscn`** становится главной сценой: внутри `Level_01 + Player`. В Level появляется контейнер `Spawned` — World слушает EventBus и кладёт атаки туда. Imp переезжает в Level (т.к. он часть уровня). |

## Как пользоваться этой серией

- Если ведёшь занятие по основной серии — `refactor/*` лежит «в кармане»
  для самостоятельных учеников и для домашнего задания «найди разницу».
- На уроке 6 (наивный) можно показать `git diff lesson-6 refactor/lesson-6`
  и обсудить, что именно поменялось и почему.
- Для разработки реальной игры дальше — продолжай от `refactor/lesson-6`,
  не от `lesson-6`. Иначе долги наивной версии будут мешать.

## Чек-лист правильной архитектуры

- [ ] Всё, что двигается через `velocity` — в `_physics_process`.
- [ ] У всех числовых параметров — типы и `@export`.
- [ ] Поиск по группе (`get_first_node_in_group`) — только в `_ready()`.
- [ ] Ни одного `get_parent().add_child(...)` через два узла вверх.
- [ ] Player живёт в `World`. Враги и временные спавны — в `Level`.
- [ ] Именованные слои коллизий, группа `player` у игрока.
- [ ] Один `EventBus` autoload — единственное «глобальное» место.
- [ ] Каждый урок = понятный коммит; каждая ветка `refactor/lesson-N`
      опирается на предыдущую.
