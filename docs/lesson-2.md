# Занятие 2 — Уровень из тайлов (TileMap)

> Прежде чем читать, убедись, что закрыл `lesson-1.md`. Мы строим поверх того,
> что было: герой уже ходит по WASD.

## Что узнаем (теория)

### Что такое тайл и тайлсет
- **Тайл** — маленький квадратик-кусочек карты (16×16 пикселей).
- **Тайлсет (TileSet)** — таблица всех тайлов из одной картинки-атласа.
  В Godot тайлсет — это **ресурс** (`.tres` или встроенный в сцену).
- Из тайлов как из лего собирают пол, стены, лестницы. Один и тот же тайл
  можно поставить хоть 1000 раз — он рисуется быстро.

### TileMapLayer (новый узел Godot 4.4+)
Раньше был один `TileMap` на много слоёв. Сейчас **каждый слой — отдельный
узел `TileMapLayer`**. Нам пока нужен один слой `Layer0` для пола.

### Сцена-уровень
Хорошая практика — каждый уровень это **отдельный .tscn файл**. Это значит:
- На уроке 4 можно сделать `level_02.tscn`, не трогая первый.
- Уровень можно «инстансировать» внутрь другой сцены.

### Куда положить Player на этом занятии
Сегодня — **внутрь `level_01.tscn`** (как ребёнок в комнате). На уроке 4
мы это переделаем: Player вынесем уровнем выше, в `World`. Но сейчас рано
объяснять «зачем», поэтому делаем как проще.

## Что делаем (практика)

### Шаг 1. Скачать/положить атлас тайлов
Скачать `0x72_DungeonTilesetII_v1.7.zip` (бесплатный CC0).
Распаковать в `assets/packs/0x72_DungeonTilesetII_v1.7/`.
Нам нужен файл `atlas_floor-16x16.png`.

### Шаг 2. Создать сцену уровня
1. New Scene → **Node** (просто Node, не Node2D — внутри уже будет TileMapLayer).
2. Переименовать в `Level_01`.
3. Сохранить как `src/levels/level_01/level_01.tscn`.

### Шаг 3. Добавить TileMapLayer
1. К `Level_01` добавить дочерний узел **TileMapLayer**, имя `Layer0`.
2. В инспекторе у Layer0 → Tile Set → New TileSet.
3. В нижней панели TileSet → Sources → `+` → Atlas → перетащить
   `atlas_floor-16x16.png`. Сетка 16×16.
4. Кнопка «Setup» нарежет атлас, поставит галочки у всех тайлов.

### Шаг 4. Нарисовать пол
1. Выбрать панель **TileMap** (внизу), выбрать любой тайл пола.
2. Зажать ЛКМ и закрасить прямоугольник 30×20 тайлов — это будет наша
   комната.
3. Сохранить.

### Шаг 5. Положить Player в уровень
1. В Level_01 → правой кнопкой → **Instantiate Child Scene** → выбрать
   `player.tscn`.
2. Поставить Player где-нибудь в центре комнаты.

### Шаг 6. Главная сцена
Project Settings → Application → Run → **Main Scene = `level_01.tscn`**.
Раньше было `player.tscn` — теперь главным становится уровень, а Player
живёт внутри него.

### Шаг 7. Запуск
F5 — герой ходит по нарисованному полу. Камера едет за ним.

Коммит:
```
git add .
git commit -m "lesson-2: tilemap level"
```

## Что мы поняли по архитектуре
- **Сцена внутри сцены** — это нормально. Player и Level — два независимых
  «кубика», их можно собирать вместе как угодно.
- **Один уровень = одна сцена** = один файл. Не пихаем все уровни в один файл.

## Подводные камни
- Если Player проваливается под пол: он не должен быть **под** TileMapLayer
  по дереву узлов и в z_index не отрицательный. Просто положи Player
  братом Layer0 — Godot нарисует тех, кто ниже в дереве, поверх.
- Тайлы видны в редакторе, но нет в игре → не выбран main scene или
  забыл сохранить уровень.
- При перемещении атласа тайлы становятся розовыми → не удаляй файлы
  ассетов через Finder, делай это через дерево Godot.
